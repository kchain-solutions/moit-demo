## Description

Evaluate whether `TWIN_TENANT_ENABLED=true` allows running Alpha and Beta as tenants on a single TWIN Node instance. This would simplify local development and CI/CD.

### What to test

1. Can two tenants have independent identities (separate DIDs)?
2. Can two tenants have independent consignment stores?
3. Does cross-tenant data isolation hold?
4. Is tenant context passed via JWT or HTTP header?

### Decision

If multi-tenant works for dev, use it for local development (single Docker container). Production demo should still use two separate nodes for realistic two-country simulation.

### Acceptance criteria

- Document: multi-tenant viability assessment (yes/no with rationale)
- If viable: Docker Compose variant with single multi-tenant node
