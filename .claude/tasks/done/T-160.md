## Description

Split the 1054-line `server/index.js` monolith into modular files. All existing API behavior must be preserved exactly (no functional changes). This is a pure structural refactoring.

### Target structure

```
server/
  index.js              # App assembly: load config, create app, mount routes, start server (~60 lines)
  config.js             # Config loader (unchanged)
  store.js              # In-memory store with accessor methods (Maps for O(1) lookups)
  proxy.js              # Routing proxy (unchanged)
  routes/
    auth.js             # POST /api/login
    config.js           # GET /api/config
    node.js             # GET /api/node, /api/node/discover, POST /api/node/connect, /api/node/disconnect
    orgs.js             # GET /api/orgs, /api/orgs/all, PUT /api/orgs/:id, POST /api/orgs/:id/register, /api/orgs/validate-credential
    consignments.js     # GET/POST /api/consignments
    documents.js        # GET/POST /api/documents, GET /api/documents/:id/download, /api/documents/:id/xml
    permissions.js      # GET /api/permissions/:id, POST /api/permissions/share, /api/permissions/revoke
    finance.js          # payments, LC, smart contracts, finance permissions (all /api/payments, /api/lc, /api/contracts, /api/finance-permissions)
    ledger.js           # GET /api/ledger, GET /api/peer/orgs
  generators/
    pdf.js              # PDF document generation (lines 98-212)
    xml.js              # XML document generation (lines 214-394)
  seed.js               # Consignment and finance seeding (lines 397-496, 956-1042)
  ws.js                 # WebSocket server + P2P peer logic (lines 879-953)
```

### Current file breakdown (line ranges in server/index.js)

- CLI args & config: 1-23
- Utility functions (genId, genHash, genDID, now): 42-45
- Credential validation: 48-64
- Store initialization: 67-79
- Ledger management: 81-96
- PDF generation: 98-212
- XML generation: 214-394
- Consignment seeding: 397-496
- Express setup + 27 route handlers: 498-877
- WebSocket server: 879-953
- Finance seeding: 956-1042
- Server start: 1044-1054

### Key constraints

- `store.js` must export the store object and accessor methods so all route files can import it
- `ws.js` must receive a reference to the HTTP server and the store
- `seed.js` must run after store initialization, receiving store and config
- Route files use `express.Router()` and are mounted in `index.js`
- Utility functions (genId, genHash, genDID, now) go in a `server/utils.js` or in `store.js`
- Credential validation logic goes in `routes/orgs.js` (only used there)

### Acceptance criteria

- `npm run demo` starts and works identically to before
- All 27 API endpoints return the same responses
- WebSocket P2P sync works between nodes
- Both `vietnam-us.json` and `adapt-africa.json` configs work
- `server/index.js` is under 80 lines
- No route handler logic remains in `server/index.js`
