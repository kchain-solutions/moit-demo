## Description

Set up Vitest as the test runner and write baseline tests for all server API routes. The project already uses Vite, so Vitest integrates with zero configuration.

### Setup

1. Add `vitest` to devDependencies
2. Add `"test": "vitest run"` and `"test:watch": "vitest"` scripts to `package.json`
3. Create `vitest.config.js` if needed (should work with existing `vite.config.js`)
4. Create `tests/server/` directory structure mirroring `server/routes/`

### Test structure (mirrors T-160 route modules)

```
tests/
  server/
    auth.test.js          # Login success, login failure (wrong password, unknown org)
    orgs.test.js          # GET orgs, PUT org, DID registration, credential validation
    consignments.test.js  # GET consignments, POST consignment, filtering
    documents.test.js     # POST document upload, GET download, GET xml
    permissions.test.js   # Share consignment, revoke consignment, permission check
    finance.test.js       # Payments CRUD, LC lifecycle, smart contract lifecycle, finance permissions
    ledger.test.js        # GET ledger, filtering by type/action
    node.test.js          # GET node info, discover, connect/disconnect
    config.test.js        # GET /api/config returns config
  helpers/
    setup.js              # Create test app instance with test config, reset store between tests
```

### Test approach

- Use Vitest's built-in `fetch` or add `supertest` for HTTP testing against the Express app
- Each test file imports the Express app (not starting the server, just the app instance)
- `helpers/setup.js` creates a fresh app + store for each test suite using a test config
- Tests run against `vietnam-us.json` config by default
- Each route must have at minimum: 1 success case, 1 error/edge case

### Minimum coverage targets

| Route module | Success tests | Error tests |
|-------------|---------------|-------------|
| auth | login OK | wrong password, unknown org |
| orgs | list orgs, update org | register DID (already registered) |
| consignments | list, create | create with missing fields |
| documents | upload, download, xml | download nonexistent |
| permissions | share, revoke | share already shared |
| finance | create payment, update status, LC lifecycle, contract lifecycle | invalid status transition |
| ledger | list, filter | empty result |
| node | info, discover | connect/disconnect |
| config | returns config | - |

### Acceptance criteria

- `npm test` runs and passes
- Every route handler in `server/routes/` has at least one test
- Tests run in under 10 seconds
- Tests do not start a real HTTP server (use supertest or app instance)
- Tests reset state between suites (no test interdependence)
- CI-ready: `npm test` exits with code 0 on success, non-zero on failure
