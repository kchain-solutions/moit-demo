## Description

`POST /api/orgs/:id/register`: In `in-process` mode, calls `adapter.createDid(orgName)` to create real `did:iota:testnet:0x...` on IOTA testnet.

### Acceptance criteria

- Org registration in `in-process` mode creates a real IOTA DID
- DID stored in org record and returned to frontend
- Explorer URL is valid and resolves to the DID document
- In `simulated` mode, behavior identical to current (fake DID)
