## Description

Copy and adapt `twin-etd-poc/apps/etd-rest-server/src/setupTwinConnectors.ts` to `server/twin/connectors.ts`.

### What to keep

- VaultConnector setup (in-memory, stores BIP-39 mnemonics)
- WalletConnector setup (IOTA testnet, faucet auto-funding via `ensureBalance()`)
- IdentityConnector setup (DID creation, Ed25519 verification methods)
- IdentityResolverConnector (DID resolution from on-chain)
- `createDid(partyName)` function (mnemonic > wallet > fund > DID > Ed25519 key)
- `resolveDid(did)` function
- Per-party registry (`partyByDid` map)
- `getPartySecretKey(did)` for signing operations

### What to remove

- NFT connector and related imports (not needed for Phase 2, no ETD workflow)
- File-based NFT deployment state persistence
- Hierarchies/accreditation integration
- ETD-specific entity storage setup

### What to adapt

- Network config: keep testnet hardcoded for now
- Faucet URL: use `https://faucet.testnet.iota.cafe`
- Explorer URL generation for DIDs
- Export functions matching `ITwinAdapter` interface

### Source reference

`twin-etd-poc/apps/etd-rest-server/src/setupTwinConnectors.ts` (714 lines, commit `542e5cf`)

### Acceptance criteria

- `initializeTwinConnectors(mnemonic)` initializes vault, wallet, identity
- `createDid("TNG Corporation")` returns real `did:iota:testnet:0x...`
- `resolveDid(did)` returns DID document from chain
- Wallet auto-funded from faucet (no manual funding)
- `isTwinInitialized()` returns true after successful init
