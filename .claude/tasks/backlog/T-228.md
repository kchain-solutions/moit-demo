## Description

Copy and adapt `twin-etd-poc/apps/etd-rest-server/src/iotaNotarizationHelper.ts` to `server/twin/notarization.ts`.

### What to keep

- `createLockedNotarization()` with builder pattern and retry logic
- `verifyNotarization()` for on-chain digest comparison
- Lazy `IotaClient` initialization
- `getChainIdentifier()` monkey-patch workaround

### What to adapt

- Metadata in `stateData`: use moit-demo document fields (`docId`, `docType`, `hash`)
- Error handling: log clearly for demo debugging
- Use SHA-256 from existing `genHash()` utility instead of Blake2b

### Source reference

`twin-etd-poc/apps/etd-rest-server/src/iotaNotarizationHelper.ts` (191 lines, commit `542e5cf`)

### Acceptance criteria

- `createLockedNotarization(hash, docId, typeCode, issuerDid, signerKey)` returns on-chain object ID
- `verifyNotarization(notarizationId, expectedHash)` returns boolean
- Retry logic handles testnet unreliability (3 retries with exponential backoff)
- Works on both macOS ARM and Linux x64 (validated by T-229)
