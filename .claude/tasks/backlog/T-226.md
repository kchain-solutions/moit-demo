- **Notes:** Created per ADR-001. The TWIN connector code from twin-etd-poc is TypeScript. The moit-demo server is plain JS. This task adds a TypeScript compilation step for the `server/twin/` directory only.
- **Section:** 2A. TWIN Connector Setup

## Description

Add TypeScript compilation for `server/twin/*.ts` files. These contain the adapted TWIN connector code from twin-etd-poc.

### Steps

1. Add `typescript` and `tsx` to devDependencies
2. Add IOTA/TWIN packages to dependencies (see ADR-001 section 8 for full list):
   - `@iota/iota-sdk`, `@iota/notarization`, `@iota/iota-interaction-ts`
   - `@twin.org/core`, `@twin.org/crypto`, `@twin.org/identity-connector-iota`
   - `@twin.org/wallet-connector-iota`, `@twin.org/vault-connector-entity-storage`
   - Plus entity-storage-models, identity-models, vault-models, wallet-models, standards-w3c-did
3. Create `server/twin/tsconfig.json` targeting ESM output
4. Add `npm run build:twin` script that compiles `server/twin/*.ts` to JS
5. Verify `server/twin/connectors.js` and `server/twin/notarization.js` are importable from the main server

### Acceptance criteria

- `npm run build:twin` compiles TypeScript without errors
- Compiled JS files are importable from `server/index.js` (ESM imports)
- IOTA/TWIN packages install successfully
- WASM bindings load without errors (see T-229)
