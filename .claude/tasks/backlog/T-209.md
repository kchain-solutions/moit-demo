## Description

`ADAPTER_MODE=real|simulated|entity-storage`. Simulated = fallback to demo server logic. Real = call TWIN Node with IOTA connectors. Entity-storage = call TWIN Node with mock connectors (no DLT, useful for development).

The TWIN Node connector architecture natively supports this concept:
- `TWIN_IDENTITY_CONNECTOR=entity-storage` = mock DIDs (no on-chain)
- `TWIN_IDENTITY_CONNECTOR=iota` = real DIDs on IOTA testnet/mainnet
- Same pattern for notarization, wallet, and other services

The adapter should read `ADAPTER_MODE` and select the appropriate route handler (demo server pass-through vs. TWIN Node client with the configured connector).
