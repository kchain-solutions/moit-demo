## Description

`server/adapter/twin-client.js` with typed methods: `login()`, `createIdentity()`, `issueCredential()`, `createNotarization()`, `uploadBlob()`, `getAuditTrail()`.

### TWIN Node REST API Mapping (from twin-node-reference.mdx v0.0.3-next.37)

| Client Method | TWIN Node Endpoint | Notes |
|---------------|-------------------|-------|
| `login(identity, password)` | `POST /authentication/login/create` | Returns JWT in httpOnly cookie (`access_token`) |
| `createIdentity()` | `POST /identity` | Returns real `did:iota:testnet:0x...` |
| `resolveIdentity(did)` | `GET /identity/{did}` | Resolves DID document |
| `issueCredential(did, claims)` | `POST /identity/{did}/credentials` | W3C Verifiable Credential issuance |
| `createNotarization(hash)` | `POST /notarization` | SHA-256 as `Notarization<vector<u8>>` Move object |
| `uploadBlob(data)` | `POST /blob-storage` | Encrypted storage (ChaCha20Poly1305) |
| `getAuditTrail()` | `GET /aig` | Auditable Item Graph entries |
| `createConsignment(data)` | `POST /supply-chain/consignments` | Maps to `ISupplyChainAppConsignment` |
| `getConsignment(id)` | `GET /supply-chain/consignments/{id}` | Returns `IConsignmentView` |

Consider using `@twin.org/supply-chain-rest-client` if the package is available on npm.
