## Description

`npm run build` emits: *Some chunks are larger than 500 kB after minification* — `server/public/assets/index-*.js` is ~521 kB (gzip ~156 kB) in a single chunk.

Options to evaluate (pick what gives best signal-to-noise for a demo):

1. **Code-split heavy views** with dynamic `import()` and `React.lazy` for routes that are not part of the initial dashboard render (e.g. `TangleExplorer`, `TradeFinance`, `Payments`, `Network` map with `react-simple-maps`).
2. **Vendor chunk splitting** via `build.rollupOptions.output.manualChunks` in `vite.config.js` (separate `react`, `react-dom`, `react-simple-maps`, `lucide-react`).
3. **Stopgap:** raise `build.chunkSizeWarningLimit` in `vite.config.js` if option 1/2 are not worth the effort for the demo.

Acceptance: `npm run build` no longer prints the 500 kB warning, and the initial JS payload is meaningfully smaller than today's 521 kB.
