## Description

Convert all inline-style modals in TradeFinance.jsx and Payments.jsx to use the shared `Modal` component created in T-161. This fixes the mobile UX inconsistency where these modals render as centered overlays instead of bottom sheets on phones.

### Current state

Two modal patterns coexist:

**Pattern A - CSS class-based (correct, has mobile bottom-sheet):**
Used in: Dashboard, Consignments, Identity, Network
```jsx
<div className="modal-bg" onClick={onClose}>
  <div className="modal" onClick={e => e.stopPropagation()}>
```

**Pattern B - Inline style-based (broken on mobile, no bottom-sheet):**
Used in: TradeFinance (3 modals), Payments (2 modals)
```jsx
<div style={{ position: 'fixed', inset: 0, background: 'rgba(0,0,0,0.5)', ... }}>
  <div className="card modal-card" style={{ width: 520, ... }}>
```

### Modals to convert

**TradeFinance.jsx:**
1. LC detail modal (view letter of credit details)
2. Create LC modal (new letter of credit form)
3. Create smart contract modal (new contract form)

**Payments.jsx:**
1. Payment detail modal (view payment details)
2. Create payment modal (new payment form)

### Work

- Replace each inline-style modal with the shared `<Modal>` component from T-161
- Preserve all modal content and functionality exactly
- Ensure `width` prop matches the current hardcoded width for each modal
- Test that close-on-backdrop-click works
- Test that Escape key closes the modal (if the shared Modal supports it)

### Acceptance criteria

- All 5 modals in TradeFinance and Payments use the shared Modal component
- No inline `position: fixed` modal overlays remain in these files
- On mobile viewport (< 768px), all 5 modals render as bottom sheets
- On desktop, modals render centered as before
- All modal functionality (forms, buttons, data display) works identically
- `npm run build` succeeds
