## Description

Implement the DSP data plane for push transfers between TWIN Nodes.

### W3C Standards involved

1. **W3C Activity Streams:** JSON-LD data model for "activities of interest." Each Activity includes generator, actor, action type (Add, Request, Create), and associated object/target.

2. **W3C ActivityPub:** Inbox (received activities) and outbox (generated activities) endpoints.

3. **W3C Linked Data Notifications:** Webhook delivery via HTTP POST to inbox endpoints.

### Implementation

- Configure `TWIN_DATASPACE_DATA_PLANE_PATH` for entity access
- Implement activity inbox handler for receiving consignment/document share activities
- Map current WebSocket `SHARE_CONSIGNMENT` message to DSP Activity of type `Add` with target consignment
- Map `REVOKE_CONSIGNMENT` to Activity of type `Remove`

### Acceptance criteria

- Push transfers deliver consignment data to peer node via HTTP POST (not WebSocket)
- Activities are JSON-LD compliant with W3C Activity Streams vocabulary
- Inbox/outbox endpoints are functional
