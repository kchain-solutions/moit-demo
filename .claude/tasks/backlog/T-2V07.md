## Description

Templates: US CBP origin verification, EU Customs read, MOIT monitoring, data sovereignty constraint (hashes only cross borders).

### Reference: TWIN ODRL EcosystemPolicy (from twin-supply-chain-reference.mdx)

The TWIN platform extends W3C ODRL with `EcosystemPolicy` (type: `https://schema.twindev.org/odrl/v1/EcosystemPolicy`). Key features:
- `PartyCollection` with `twin:jsonpath:$` refinement (match on party role, e.g., `BorderAgency`)
- Target refinement with JSONPath expressions over `https://vocabulary.uncefact.org/Consignment` objects
- Geographic targeting via `departureEvent` / `arrivalEvent` location country codes
- `obligation` action: `inform` with assignee (e.g., border agency DID)

ISN Notify Template is the reference pattern: it creates an obligation to inform a border agency when consignment departure/arrival matches a country code (e.g., `VN` for Vietnam, `US` for United States).

Vietnam-specific templates should use:
- Port codes: `VNSGN` (Ho Chi Minh), `VNHPH` (Hai Phong)
- Country code: `VN` for departure, `US`/`NL`/`DE`/`JP` for arrival
- Roles: `BorderAgency`, `CustomsAuthority`, `RegulatoryAuthority`
